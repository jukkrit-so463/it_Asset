## การใช้งาน Modal

### เพิ่มข้อมูลใหม่
1. คลิกปุ่ม "เพิ่ม..." ในแต่ละหน้า
2. กรอกข้อมูลในฟอร์ม
3. คลิก "เพิ่ม" เพื่อบันทึก
4. ระบบจะแสดง toast notification

### แก้ไขข้อมูล
1. คลิกปุ่ม "แก้ไข" ในแถวของข้อมูล
2. ข้อมูลจะถูกโหลดในฟอร์ม
3. แก้ไขข้อมูลที่ต้องการ
4. คลิก "อัปเดต" เพื่อบันทึก

### ลบข้อมูล
1. คลิกปุ่ม "ลบ" ในแถวของข้อมูล
2. ระบบจะแสดง Dialog ยืนยัน
3. คลิก "ยืนยัน" เพื่อลบข้อมูล
4. ระบบจะแสดง toast notification

## การเชื่อมต่อ API

### Assets API
- `GET /api/assets` - ดึงรายการครุภัณฑ์
- `POST /api/assets` - เพิ่มครุภัณฑ์ใหม่
- `PUT /api/assets/:id` - แก้ไขครุภัณฑ์
- `DELETE /api/assets/:id` - ลบครุภัณฑ์

### Users API
- `GET /api/users` - ดึงรายการผู้ใช้งาน
- `POST /api/users` - เพิ่มผู้ใช้งานใหม่
- `PUT /api/users/:id` - แก้ไขผู้ใช้งาน (ต้องเพิ่ม)
- `DELETE /api/users/:id` - ลบผู้ใช้งาน (ต้องเพิ่ม)

### Locations API
- `GET /api/locations` - ดึงรายการสถานที่
- `POST /api/locations` - เพิ่มสถานที่ใหม่
- `PUT /api/locations/:id` - แก้ไขสถานที่ (ต้องเพิ่ม)
- `DELETE /api/locations/:id` - ลบสถานที่ (ต้องเพิ่ม)

### Success Feedback
- แสดง toast success เมื่อดำเนินการสำเร็จ
- แสดงข้อความยืนยันการดำเนินการ

## การทดสอบ

### ข้อมูลตัวอย่าง
หลังจากรัน `npm run seed-data` จะมี:
- **5 สถานที่**: ฝ่ายผลิต, บัญชี, บุคคล, ไอที
- **5 ผู้ใช้งาน**: admin, user1, user2, user3, itadmin
- **10 ครุภัณฑ์**: Computer, Printer, Scanner, Server, Network Switch

### การทดสอบฟีเจอร์
1. **เพิ่มข้อมูล**: ทดสอบเพิ่มครุภัณฑ์, ผู้ใช้งาน, สถานที่
2. **แก้ไขข้อมูล**: ทดสอบแก้ไขข้อมูลที่มีอยู่
3. **ลบข้อมูล**: ทดสอบลบข้อมูลพร้อมยืนยัน
4. **ค้นหา**: ทดสอบการค้นหาและกรองข้อมูล
5. **Dashboard**: ทดสอบการแสดงสถิติจริง
